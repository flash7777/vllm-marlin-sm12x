# UF19v5: System-Scope Minimal Proxy AllReduce

## Zusammenfassung

UF19v5 ersetzt den NCCL AllReduce (18 µs) durch einen minimalen Proxy mit
system-scope PTX Memory-Order-Instruktionen. Ziel: **~6.5 µs/call** (64% schneller).

## Versionshistorie

| Version | Ansatz | Ergebnis |
|---------|--------|----------|
| v1-v2 | Frühe Experimente | — |
| v3 | NCCL Net Plugin (libnccl-net-uf19.so) | Kein Latenzgewinn |
| v4 | Custom RDMA (ibverbs, eigene GPU-Kernels) | L2-Kohärenzproblem auf GB10 |
| **v5** | **System-scope PTX + minimaler CPU-Proxy** | **3.5 µs lokal, ~6.5 µs mit Wire** |

## Durchbruch: System-Scope PTX Memory Ordering

### Das Problem (v4)
GB10 Unified Memory: GPU L2 Cache ist NICHT kohärent mit NIC RDMA DMA.
- `ld.global.cv` (cache volatile): Funktioniert NICHT auf SM121
- `__threadfence_system()` + normale Loads: GPU liest stale L2
- `cudaMemcpyAsync`: Funktioniert, aber 4.6 µs CUDA-API-Overhead pro Call

### Die Lösung (v5)
CUDA 13 / SM121 PTX Memory-Order-Qualifier:
```
st.release.sys.global.b32   ← GPU Store → sofort für CPU/NIC sichtbar
ld.acquire.sys.global.b32   ← GPU Load  → invalidiert stale L2, liest aus DRAM
```

Diese Instruktionen laufen direkt im GPU-Kernel (kein CUDA-API-Call) und sind
CUDA-Graph-kompatibel.

### Benchmark-Beweis

| Methode | GPU→CPU→GPU (8KB) | Overhead |
|---------|-------------------|----------|
| `cudaMemcpyAsync` D2H+H2D | 9.3 µs | 4.4 µs fixed API cost |
| `st/ld.sys` (Kernel) | ~3 µs | 0 µs (direkt im Kernel) |
| System-scope atomic RTT | 1.41 µs | — |

| Proxy-Simulation (kein Netzwerk) | Latenz |
|----------------------------------|--------|
| cudaMemcpy-Proxy | 15.4 µs |
| sys-scope Proxy (v5) | **3.5 µs** |
| + Wire (RoCE ~3 µs) | **~6.5 µs** |
| NCCL AllReduce (Referenz) | 18.0 µs |

## Architektur

```
GPU Kernel (graph-capturable, 1 Launch):
  ┌─────────────────────────────────────────────────┐
  │ 1. Wait send_done (prev proxy complete)         │
  │ 2. st.release.sys input → pinned send_buf       │
  │ 3. atomicExch_system(send_flag, step)            │
  │ 4. Poll recv_flag (atomicAdd_system)             │
  │ 5. ld.acquire.sys recv_buf → reduce + output     │
  └─────────────────────────────────────────────────┘
           ↕ send_flag/recv_flag (cudaHostAlloc, system-scope atomics)
  ┌─────────────────────────────────────────────────┐
  │ CPU Proxy Thread (background):                   │
  │ 1. Poll send_flag (__atomic_load)               │
  │ 2. ibv_post_send(send_buf → peer recv_buf)      │
  │    + ibv_post_send(flag → peer recv_flag)        │
  │ 3. ibv_poll_cq (wait for NIC)                   │
  │ 4. __atomic_store(send_done, step)              │
  └─────────────────────────────────────────────────┘
           ↕ RDMA WRITE (RoCE, ConnectX-7)
  ┌─────────────────────────────────────────────────┐
  │ Remote: NIC schreibt direkt in recv_buf + flag   │
  │ (cudaHostAlloc memory, ibv_reg_mr registriert)  │
  └─────────────────────────────────────────────────┘
```

### Schlüsselunterschiede zu v4

| | v4 | v5 |
|---|---|---|
| Send-Pfad | `copy_to_send` (normale Stores) | `st.release.sys` (kohärent) |
| Recv-Pfad | `ld.global.cv` (BROKEN auf SM121) | `ld.acquire.sys` (funktioniert!) |
| Buffers | `posix_memalign` (Heap) | `cudaHostAlloc` (Pinned) |
| GPU Kernels | 5 separate Launches | 1 persistenter Kernel |
| Kohärenz | Kaputt (L2 stale reads) | Korrekt (PTX memory order) |

### Warum cudaHostAlloc statt posix_memalign?

- `cudaHostAlloc` Speicher ist page-locked (pinned) → stabile DMA-Performance
- `ibv_reg_mr` funktioniert auf pinned Host-Memory (kein nvidia-peermem nötig)
- System-scope PTX-Instruktionen funktionieren auf beiden Allokationstypen
- Aber: pinned Memory hat konsistentere Latenz für NIC DMA

## Build

```bash
nvcc -O2 -arch=sm_121 -shared -Xcompiler -fPIC \
    -o /opt/vllm/libuf19v5_sys.so uf19v5_sys.cu \
    -L/usr/lib/aarch64-linux-gnu -libverbs -lpthread
```

## Env-Variablen

```bash
VLLM_UF_UF19V5=1              # UF19v5 aktivieren
VLLM_UF19_PEER_IP=192.168.0.116  # Peer RDMA IP
VLLM_UF19_IB_DEV=rocep1s0f0      # IB Device
VLLM_UF19_GID_IDX=-1             # GID Index (-1=auto)
UF19V5_LIB_PATH=/opt/vllm/libuf19v5_sys.so
```
